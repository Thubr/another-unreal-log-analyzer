# Security Policy

## Supported Versions

This project is in active development. Security fixes are applied on the default branch.

## Reporting a Vulnerability

Please do **not** open public issues for security vulnerabilities.

Instead, report vulnerabilities privately to project maintainers with:

- a clear description of the issue,
- reproduction steps,
- impact assessment,
- and any suggested mitigations.

Maintainers will acknowledge receipt and provide follow-up status updates.

## Public Repository Data Safety

This repository is public from day one.

Do **not** commit:

- real game logs,
- proprietary or NDA-covered platform/SDK details,
- real player/service/session/ticket/lobby identifiers,
- real IPs, hostnames, machine names, internal paths,
- secrets, tokens, or credentials.

Use synthetic fixtures by default. Sanitized examples are only allowed when explicitly reviewed and confirmed safe.
